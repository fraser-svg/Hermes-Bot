# Strategy: Build First, Pitch Never

## Core Shift
Old model: find problem -> pitch fix -> ask for payment.
New model: find problem -> build fix -> show completed asset -> sell ongoing operation.

## Why It Converts
- Reciprocity: tangible asset > audit.
- Endowment effect: prospect sees branded site as already theirs.
- Loss aversion: removing demo feels like loss.
- Low cognitive load: no imagination required.
- Demonstration over claims: proof replaces promises.

## Phase-1 Execution (Current Vector)
Niche: Electricians
City: Edinburgh

1) Detect prospects (Google Maps via prospect.py)
   - Primary: no website
   - Expansion lane: poor website quality (v2 scoring)

2) Build first (generate.py only)
   - Build complete demo site before outreach
   - Keep hosted on controlled infra
   - Not connected to domain/indexing/analytics until retained

3) Human outreach (manual approval gate)
   - Short message + live demo link
   - Show old vs new delta
   - Ask: "Want me to put it live?"

4) Close into retainer
   - Entry: hosting/updates/GBP optimization
   - Upsell: ads + tracking + full growth stack

## Asset Bundle Per Prospect
- Website demo
- Website quality audit evidence (v2 rubric)
- GBP improvement notes
- Messaging rewrite from reviews
- One follow-up sequence draft

## Guardrails
- Website generation stays pipeline-only: generate.py + Gemini 3.1 Pro
- No auto-send outreach in Phase 1
- One city + one niche until KPI gates met

## KPI Targets
- 30 qualified leads
- 10 built sites (15/15 validation + readability pass)
- 10 approved outreaches
- >=1 conversion

## Positioning Language
"Built this for you first so you can see the fix, not imagine it. If you want it live, I’ll run it end-to-end."
